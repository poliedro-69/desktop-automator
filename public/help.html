1. Inicio rápido

  1. Añadir actividades: Click en + Añadir actividad y selecciona de la lista por categorías.
  2. Configurar parámetros: Expande cada actividad con click para ver sus opciones.
  3. Ordenar: Usa las flechas ↑↓ a la izquierda de cada actividad para reordenar.
  4. Activar/Desactivar: El toggle de cada actividad controla si se ejecuta o se salta.
  5. Iniciar: Pulsa ▶ Iniciar automatización para comenzar.
  6. Detener: Pulsa ⏹ Detener en cualquier momento.
La automatización se ejecuta secuencialmente: cada actividad espera a que la anterior termine antes de continuar.

4. Variables y plantillas

Cualquier campo de texto en los parámetros acepta variables dinámicas con la sintaxis {{nombre}}. Se resuelven justo antes de ejecutar cada actividad.

Variables disponibles

VariableValorEjemplo
{{date}}Fecha actual2026-07-03
{{time}}Hora actual14:30:22
{{timestamp}}Fecha y hora2026-07-03 14:30:22
{{counter}}Nº de actividad en el ciclo1, 2, 3...
{{cycle}}Nº de ciclo actual1, 2...
{{prev_result}}Resultado de la actividad anterior(texto variable)
{{clipboard}}Último output de PowerShell(texto variable)
{{random_id}}ID aleatorio de 8 caracteresa7x3k9m2
{{random_number}}Número aleatorio 1-99994217
{{custom.X}}Variable definida con "Definir variable"lo que asignes

Ejemplo de uso

Actividad: Abrir archivo
Ruta: C:\Informes\reporte_{{date}}.xlsx

→ Se resuelve a: C:\Informes\reporte_2026-07-03.xlsx

Encadenamiento de resultados

El resultado de cada actividad se guarda automáticamente en {{prev_result}}. Esto permite que una actividad use la salida de la anterior:

1. ⚡ PowerShell → script: "Get-Date -Format 'dd-MM-yyyy'"
   → resultado: "03-07-2026"
   
2. ✍️ Escritura → text: "Fecha de hoy: {{prev_result}}"
   → escribe: "Fecha de hoy: 03-07-2026"

5. Perfiles

Los perfiles guardan toda tu configuración (actividades + ajustes globales) en un archivo .json.

  • Guardar: Botón "Guardar" en la barra superior → diálogo para elegir dónde guardarlo
  • Abrir: Botón "Abrir" → seleccionar un archivo de perfil existente
  • Recientes: Icono 🕐 → lista de los últimos 8 perfiles usados (click para cargar)
Puedes crear perfiles diferentes para cada flujo de trabajo: uno para la jornada matutina, otro para la tarde, otro para demos, etc.

6. Modo bandeja del sistema

El botón 👁️ Ocultar minimiza la app a la bandeja del sistema (system tray). La automatización sigue corriendo en segundo plano.

  • Click izquierdo en el icono del tray: Muestra/oculta la ventana
  • Click derecho → Mostrar ventana: Restaura la app
  • Click derecho → Salir: Cierra la app completamente
Si cierras la ventana con la X, la app se cierra completamente. Usa "Ocultar" para que siga en segundo plano.

7. Temas claro/oscuro

Icono ☀️/🌙 en la barra superior. El tema se guarda automáticamente y se aplica al reiniciar.

Si no eliges tema manualmente, la app usa la configuración del sistema operativo (Configuración → Personalización → Colores).

10. Preguntas frecuentes

¿Funciona con la pantalla bloqueada?

No. El mouse y teclado simulados requieren que Windows esté en una sesión activa (pantalla desbloqueada). Si bloqueas la pantalla (Win+L), las acciones de input dejarán de funcionar hasta que desbloquees.

¿Puedo usar coordenadas de un segundo monitor?

Sí. En configuración de varios monitores, el monitor secundario suele empezar en X=1920 (si el principal es 1920px de ancho). Usa coordenadas como x: 2200, y: 400 para el segundo monitor.

¿Qué pasa si la app objetivo no está abierta?

Si intentas hacer click en una coordenada o escribir texto pero la ventana no está en primer plano, la acción se ejecutará en cualquier ventana que tenga el foco en ese momento. Usa "Abrir aplicación" primero para asegurarte.

¿Los perfiles se pueden compartir entre PCs?

Sí, los archivos .json de perfil son portables. Solo asegúrate de que las rutas de archivos existan en el PC destino.

¿Cómo hago que se ejecute al iniciar Windows?

Copia un acceso directo del .exe en:

C:\Users\TU_USUARIO\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

O crea una actividad de PowerShell que lo registre.

¿Consume muchos recursos?

No. La app usa ~15 MB de RAM y CPU casi nulo cuando espera entre acciones. Está construida con Rust (backend nativo) y WebView2 (frontend ligero).
