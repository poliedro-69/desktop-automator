1. Quick start

  1. Add activities: Click + Add activity and select from the categorized list.
  2. Configure parameters: Expand each activity by clicking on it to reveal its options.
  3. Reorder: Use the ↑↓ arrows on the left side of each activity to change execution order.
  4. Enable/Disable: The toggle on each activity controls whether it runs or gets skipped.
  5. Start: Press ▶ Start automation to begin execution.
  6. Stop: Press ⏹ Stop at any time to halt.
Automation runs sequentially: each activity waits for the previous one to finish before continuing.

3. Configuration

Click the ⚙️ icon in the top toolbar to open the global settings panel:

OptionDescription
Min / Max delayRandom wait time between activities (seconds)
Repeat in loopWhen enabled, repeats the entire sequence
Cycles (0=∞)Number of repetitions. 0 means infinite until you stop it
Human micro-pausesAdds small random pauses (0.5–2s) after each action for a more natural feel
If loop is disabled the automation runs once through the activity list and stops automatically.

Usage example

Activity: Open document
Path: C:\Reports\report_{{date}}.xlsx

→ Resolves to: C:\Reports\report_2026-07-03.xlsx

Result chaining

The result of each activity is automatically stored in {{prev_result}}. This allows an activity to use the output of the previous one:

1. ⚡ PowerShell → script: "Get-Date -Format 'dd-MM-yyyy'"
   → result: "03-07-2026"
   
2. ✍️ Auto-typing → text: "Today's date: {{prev_result}}"
   → types: "Today's date: 03-07-2026"

Custom variables example

1. 📌 Set variable → name: "project", value: "ClientX"
2. 📌 Set variable → name: "version", value: "{{random_id}}"
3. ⚡ PowerShell → script: "New-Item -Path 'C:\Builds\{{custom.project}}_{{custom.version}}' -ItemType Directory"
4. 📸 Screenshot → outputPath: "C:\Builds\{{custom.project}}_{{custom.version}}"

5. Profiles

Profiles save your entire configuration (activities + global settings) into a .json file.

  • Save: "Save" button in the top toolbar → file dialog to choose where to save
  • Open: "Open" button → select an existing profile file
  • Recents: 🕐 icon → list of the last 8 used profiles (click to load)
You can create different profiles for each workflow: one for morning routines, another for afternoons, another for demos, etc.
File paths inside a profile are absolute. If you share a profile with another PC, make sure the referenced paths exist on the target machine.

6. System tray mode

The 👁️ Hide button minimizes the app to the system tray. The automation keeps running in the background.

  • Left-click the tray icon: Shows/hides the window
  • Right-click → Show window: Restores the app
  • Right-click → Exit: Closes the app completely
If you close the window with the X button, the app shuts down completely. Use "Hide" to keep it running in the background.

7. Light/dark themes

Click the ☀️/🌙 icon in the top toolbar to toggle. Your theme choice is saved automatically and restored on next launch.

If you haven't manually selected a theme, the app follows the operating system setting (Settings → Personalization → Colors).

10. Frequently asked questions

Does it work with the screen locked?

No. Simulated mouse and keyboard input requires an active Windows session (screen unlocked). If you lock the screen (Win+L), input actions will stop working until you unlock it. Non-input activities like PowerShell scripts and file operations will continue normally.

Can I use coordinates on a second monitor?

Yes. In multi-monitor setups, the secondary monitor usually starts at X=1920 (if the primary monitor is 1920px wide). Use coordinates like x: 2200, y: 400 for the second monitor. You can check actual positions in Display Settings → Monitor arrangement.

What happens if the target app isn't open?

If you try to click at a coordinate or type text but the target window isn't in the foreground, the action will execute on whatever window has focus at that moment. Use "Open application" first to ensure your target app is running, then use alt+tab or a click to bring it to front.

Can profiles be shared between PCs?

Yes. Profile .json files are portable. Just make sure that any file paths referenced in the activities exist on the target PC. You can use environment variables in PowerShell scripts for cross-machine compatibility.

How do I run it on Windows startup?

Place a shortcut to the .exe in the Windows startup folder:

C:\Users\YOUR_USER\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

Alternatively, you can use Task Scheduler for more control over when and how the app launches.

How much resources does it use?

Very little. The app uses ~15 MB of RAM and near-zero CPU while waiting between actions. It's built with Rust (native backend) and WebView2 (lightweight frontend), making it one of the most efficient automation tools available.
